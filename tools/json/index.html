JSON整形・検証

JSONテキストを貼り付けて、「整形」「圧縮」「検証」のいずれかを実行してください。 入力値はこの端末（ブラウザ）内だけで処理され、外部に送信されることはありません。

整形オプション

結果


        

      


      

        使い方の例:
        {"b":2,"a":1} を「整形」すると読みやすい複数行表示に、「圧縮」すると
        {"b":2,"a":1} のような1行表示になります。
        「キーをアルファベット順にソート」をオンにすると {"a":1,"b":2} の順で出力されます。
        「検証」では、不正なJSON（例: {"a":1,}）を入力するとエラー内容と位置を表示します。
      


    

  


  
  

    
使い方

    
    
      
  1. テキストエリアにJSONを貼り付けます。
    2. 
      
  2. 「整形」を押すと、選択したインデント幅（2または4）で見やすく整形されたJSONが表示されます。「キーをアルファベット順にソート」をオンにすると、各オブジェクトのキーがソートされた状態で出力されます。
    3. 
      
  3. 「圧縮（minify）」を押すと、改行・空白を取り除いた1行のJSONが表示されます。
    4. 
      
  4. 「検証」を押すと、JSONが正しい形式かどうかを判定します。不正な場合は、エラーの種類とおおよその位置（行・列、文字位置）を表示します。
    5. 
      
  5. 結果は「結果をコピー」ボタンでクリップボードにコピーできます。
    6. 
    

    

      入力されたJSONは、この端末（ブラウザ）内だけで処理され、外部のサーバーへ送信されることはありません。
    

  


  
  

    
よくある質問


    
「整形」と「圧縮」の違いは何ですか？

    

      「整形」は、JSONをインデント（字下げ）付きの複数行に整え、人が読みやすい形にします。
      「圧縮（minify）」は、改行や余分な空白を取り除き、データサイズを小さくした1行のJSONにします。
      用途に応じて使い分けてください。
    


    
不正なJSONを入力するとどうなりますか？

    

      「整形」「圧縮」「検証」のいずれかを押した際にJSONとして解析できない場合、
      エラーメッセージとともに、エラーが発生したおおよその位置（文字位置・行・列）を表示します。
      例えば、末尾に余分な「,」（カンマ）がある場合や、引用符の閉じ忘れなどが典型的な原因です。
    


    
キーの並び順を変えることはできますか？

    

      「キーをアルファベット順にソート」をオンにして「整形」または「圧縮」を実行すると、
      JSON内のすべてのオブジェクトのキーがアルファベット順（Unicodeコードポイント順）に並び替えられます。
      配列の順序は変更されません。
    


    
大きなJSONでも処理できますか？

    

      ブラウザのJSON.parse・JSON.stringifyを利用しているため、
      ブラウザが扱える範囲のサイズであれば処理できます。
      非常に大きなJSON（数十MB以上）では、処理に時間がかかったり、
      ブラウザの性能によっては固まる可能性があります。その場合は分割して処理することをおすすめします。
    


    
入力したJSONは保存・送信されますか？

    

      いいえ。本ツールは入力されたJSONを外部に送信しません。すべての処理はこの端末（ブラウザ）内で行われます。
      画面表示用の設定（ダーク/ライト表示等）のみ、ブラウザのlocalStorageに保存される場合があります。
